# ESP8266/ESP-12F 红外智能遥控器项目

## 项目概述

这是一个基于 ESP8266/ESP-12F 的智能红外遥控系统，可以实现：

1. **红外学习功能** - 学习并存储电视机遥控器的 NEC 编码
2. **WiFi 远程控制** - 通过手机/电脑网页远程发送红外指令
3. **NTP 时间同步** - 自动从网络获取准确时间
4. **定时任务** - 每天早上 8 点自动开机播放，晚上 7 点自动关机

## 硬件要求

- **ESP8266/ESP-12F 开发板**（如 NodeMCU、Wemos D1 Mini 等）
- **红外接收模块**（VS1838B 等，通常集成在模块上）
- **红外发射管**（通常集成在模块上）
- **USB 数据线**

### 引脚连接（默认配置）

| 功能 | GPIO | NodeMCU 引脚 |
|------|------|--------------|
| 红外接收 | GPIO14 | D5 |
| 红外发射 | GPIO4  | D2 |

> 注意：如果你的模块引脚不同，请修改代码中的 `IR_RECEIVE_PIN` 和 `IR_SEND_PIN`。

## 软件准备

### 1. 安装 Arduino IDE

从 [arduino.cc](https://www.arduino.cc/en/software) 下载并安装 Arduino IDE。

### 2. 添加 ESP8266 开发板支持

1. 打开 Arduino IDE → 文件 → 首选项
2. 在"附加开发板管理器网址"中添加：
   ```
   http://arduino.esp8266.com/stable/package_esp8266com_index.json
   ```
3. 打开 工具 → 开发板 → 开发板管理器
4. 搜索 "ESP8266" 并安装

### 3. 安装所需库

在 Arduino IDE 中，选择 **工具 → 管理库**，搜索并安装以下库：

| 库名 | 作者 | 版本 |
|------|------|------|
| **IRremoteESP8266** | David Conran | >= 2.7.0 |
| **NTPClient** | Fabrice Weinberg | >= 3.2.0 |
| **ArduinoJson** | Benoit Blanchon | >= 6.0 |

> 注意：IRremoteESP8266 是关键库，用于红外编解码。

## 上传程序

### 1. 修改 WiFi 配置

打开 `ESP8266_IR_Controller_v2.ino` 文件，找到以下行并修改：

```cpp
// WiFi 配置 - 请修改为你的网络信息
const char* ssid = "YOUR_WIFI_SSID";      // 改成你的WiFi名称
const char* password = "YOUR_WIFI_PASSWORD";  // 改成你的WiFi密码
```

### 2. 选择开发板和端口

1. 用 USB 线连接 ESP8266 到电脑
2. 在 Arduino IDE 中选择：
   - **工具 → 开发板 → ESP8266 Boards → NodeMCU 1.0** (或你的开发板型号)
   - **工具 → 端口 → 选择对应的 COM 端口**
   - **工具 → Flash Size → 4MB (FS:2MB OTA:~1019KB)**

### 3. 编译上传

1. 点击 Arduino IDE 的 **上传** 按钮（→ 图标）
2. 等待编译完成并开始上传
3. 上传完成后，打开 **工具 → 串口监视器**，波特率选择 **115200**
4. 你应该能看到 ESP8266 的启动信息

## 使用方法

### 1. 访问 Web 控制界面

1. 打开串口监视器查看 ESP8266 的 IP 地址，例如：
   ```
   WiFi已连接!
   IP地址: 192.168.1.100
   ```

2. 在手机或电脑的浏览器中访问这个 IP 地址：
   ```
   http://192.168.1.100
   ```

3. 你应该能看到红外遥控器的 Web 控制界面

### 2. 学习红外编码

1. 在 Web 界面中找到 **红外学习** 区域
2. 输入命令名称，例如：`power`（电源键）
3. 点击 **开始学习** 按钮
4. 在 30 秒内，将电视遥控器对准 ESP8266 的红外接收头，按下电源键
5. 看到"编码已保存"的提示即表示学习成功
6. 重复上述步骤学习其他按键（如 vol+, vol-, source, ok 等）

### 3. 手动发送指令

1. 在 **手动控制** 区域输入已学习的命令名称
2. 点击 **发送指令** 按钮
3. ESP8266 会发射对应的红外信号

### 4. 自动播放序列

点击 **立即执行** 按钮，ESP8266 会按顺序执行：
1. 发送电源键（开机）
2. 等待 3 秒
3. 发送信号源键
4. 发送确认键
5. 导航到视频并播放

### 5. 定时任务

系统默认配置：
- **早上 8:00** - 自动开机并播放视频
- **晚上 19:00** - 自动关机

修改定时时间：编辑代码中的以下配置：
```cpp
const int AUTO_ON_HOUR = 8;     // 早上8点
const int AUTO_OFF_HOUR = 19;   // 晚上7点
```

## 故障排除

### 问题1：上传失败，显示 "error: espcomm_open failed"

**解决方案：**
1. 检查 USB 线是否支持数据传输（有些线只支持充电）
2. 安装 CH340/CP2102 驱动程序
3. 按住 FLASH 按钮，然后按一下 RESET 按钮进入烧录模式
4. 重新选择正确的 COM 端口

### 问题2：红外学习失败，无法接收信号

**解决方案：**
1. 检查红外接收头的引脚连接是否正确
2. 确保遥控器发射端对准接收头，距离在 5-10 厘米
3. 检查遥控器电池是否有电
4. 在串口监视器中查看是否有原始信号输出

### 问题3：WiFi 连接失败

**解决方案：**
1. 检查 WiFi 名称和密码是否正确（区分大小写）
2. 确保 WiFi 是 2.4GHz 频段（ESP8266 不支持 5GHz）
3. 检查 WiFi 信号强度
4. 尝试重启路由器

### 问题4：Web 界面无法访问

**解决方案：**
1. 确保手机和 ESP8266 连接到同一个 WiFi 网络
2. 检查串口监视器中的 IP 地址是否正确
3. 尝试刷新页面或清除浏览器缓存
4. 检查是否有防火墙阻止连接

### 问题5：定时任务不准确

**解决方案：**
1. 确保 ESP8266 能正常连接到 NTP 服务器
2. 检查时区设置是否正确（`gmtOffsetSec`）
3. 可以在代码中增加调试输出查看当前时间
4. 考虑使用外部 RTC 模块作为备用时钟源

## 进阶功能

### 接入 Home Assistant

如果你想把这个遥控器接入 Home Assistant，可以：

1. 添加 RESTful Switch 配置：
```yaml
switch:
  - platform: rest
    name: "TV Power"
    resource: http://192.168.1.100/send?cmd=power
    method: post
```

2. 使用 MQTT 协议（需要额外代码支持）

```cpp
#include <ArduinoOTA.h>

void setup() {
  // ... 其他初始化代码 ...
  ArduinoOTA.begin();
}

void loop() {
  ArduinoOTA.handle();
  // ... 其他代码 ...
}
```
