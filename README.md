# ESP8266/ESP-12F 红外智能遥控器项目

## 项目概述

这是一个基于 ESP8266/ESP-12F 的智能红外遥控系统，可以实现：

### v3 版本（推荐）- 完整功能版
- **红外学习功能** - 学习并存储电视机遥控器编码，支持自定义命名
- **操作链管理** - 创建自定义操作序列（发送命令 + 等待时间）
- **定时任务** - 每天定时触发不同的操作链
- **Web 配置界面** - 全功能图形化配置

### v2 版本
- 红外学习功能 - 学习并存储电视机遥控器的 NEC 编码
- WiFi 远程控制 - 通过手机/电脑网页远程发送红外指令
- NTP 时间同步 - 自动从网络获取准确时间
- 定时任务 - 每天早上 8 点自动开机播放，晚上 7 点自动关机

### v1 版本
- 基础版本，不含 NTP 时间同步（适合简单应用）

---

## 硬件要求

- **ESP8266/ESP-12F 开发板**（如 NodeMCU、Wemos D1 Mini 等）
- **红外接收模块**（VS1838B 等，通常集成在模块上）
- **红外发射管**（通常集成在模块上）
- **USB 数据线**

### 引脚连接（默认配置）

| 功能 | GPIO | NodeMCU 引脚 |
|------|------|--------------|
| 红外接收 | GPIO14 | D5 |
| 红外发射 | GPIO4  | D2 |

> 注意：如果你的模块引脚不同，请修改代码中的 `IR_RECEIVE_PIN` 和 `IR_SEND_PIN`。

## 软件准备

### 1. 安装 Arduino IDE

从 [arduino.cc](https://www.arduino.cc/en/software) 下载并安装 Arduino IDE。

### 2. 添加 ESP8266 开发板支持

1. 打开 Arduino IDE → 文件 → 首选项
2. 在"附加开发板管理器网址"中添加：
   ```
   http://arduino.esp8266.com/stable/package_esp8266com_index.json
   ```
3. 打开 工具 → 开发板 → 开发板管理器
4. 搜索 "ESP8266" 并安装

### 3. 安装所需库

在 Arduino IDE 中，选择 **工具 → 管理库**，搜索并安装以下库：

| 库名 | 作者 | 版本 | 用途 |
|------|------|------|------|
| **IRremoteESP8266** | David Conran | >= 2.7.0 | 红外编解码 |
| **NTPClient** | Fabrice Weinberg | >= 3.2.0 | NTP 时间同步 |
| **ArduinoJson** | Benoit Blanchon | >= 6.0 | JSON 解析 |
| **LittleFS_esp8266** | ESP8266 Community | >= 0.1.0 | 文件系统 |

## 上传程序（v3 版本）

### 1. 修改 WiFi 配置

打开 `ESP8266_IR_Controller_v3.ino` 文件，确认以下配置：

```cpp
const char* ssid = "Nuance";              // WiFi 名称
const char* password = "Nuance2018";      // WiFi 密码
```

### 2. 选择开发板和端口

1. 用 USB 线连接 ESP8266 到电脑
2. 在 Arduino IDE 中选择：
   - **工具 → 开发板 → ESP8266 Boards → NodeMCU 1.0** (或你的开发板型号)
   - **工具 → 端口 → 选择对应的 COM 端口** (如 COM3)
   - **工具 → Flash Size → 4MB (FS:2MB OTA:~1019KB)**
   - **工具 → CPU Frequency → 160 MHz**

### 3. 上传文件系统（LittleFS）

v3 版本需要 LittleFS 文件系统来存储数据：

1. 在 Arduino IDE 中安装 **ESP8266 LittleFS Data Upload** 插件
2. 点击 **工具 → ESP8266 LittleFS Data Upload** (第一次会自动格式化)

> 注意：如果不想安装插件，代码会自动处理文件系统初始化。

### 4. 编译上传

1. 点击 Arduino IDE 的 **上传** 按钮（→ 图标）
2. 等待编译完成并开始上传
3. 上传完成后，打开 **工具 → 串口监视器**，波特率选择 **115200**
4. 你应该能看到 ESP8266 的启动信息

## v3 使用方法

### 1. 访问 Web 控制界面

1. 打开串口监视器查看 ESP8266 的 IP 地址，例如：
   ```
   WiFi 已连接!
   IP 地址: 192.168.1.100
   ```

2. 在手机或电脑的浏览器中访问这个 IP 地址：
   ```
   http://192.168.1.100
   ```

3. 你会看到四个标签页：**红外学习**、**编码管理**、**操作链**、**定时任务**

### 2. 学习红外编码

1. 切换到 **红外学习** 标签页
2. 输入按键名称，例如：`power`（电源键）、`vol+`（音量加）
3. 点击 **开始学习** 按钮
4. 在 30 秒内，将遥控器对准 ESP8266 的红外接收头，按下对应按键
5. 看到"学习成功"提示即表示成功
6. 重复上述步骤学习其他按键

### 3. 管理红外编码

切换到 **编码管理** 标签页可以：
- 查看所有已学习的编码
- 点击 **测试** 按钮发送红外信号
- 点击 **删除** 按钮删除不需要的编码

### 4. 创建操作链

1. 切换到 **操作链** 标签页
2. 点击 **+ 新建操作链**
3. 输入操作链名称，如"早上开机播放"
4. 添加操作步骤：
   - **发送命令**：选择已学习的按键名称
   - **等待**：输入等待毫秒数（如 1000 = 1秒）
5. 按顺序添加所有步骤（开机 → 等待 → 信号源 → 确认...）
6. 点击 **保存**
7. 可以点击 **运行** 按钮测试操作链

**示例操作链：**
```
1. 发送: power      (开机)
2. 等待: 3000       (等3秒)
3. 发送: source     (信号源)
4. 等待: 1000       (等1秒)
5. 发送: down       (向下)
6. 等待: 500        (等0.5秒)
7. 发送: down       (向下)
8. 等待: 500        (等0.5秒)
9. 发送: ok         (确认)
```

### 5. 设置定时任务

1. 切换到 **定时任务** 标签页
2. 点击 **+ 新建定时任务**
3. 设置触发时间（小时和分钟）
4. 选择要执行的操作链
5. 点击 **保存**
6. 可以通过开关启用/禁用定时任务

**示例定时任务：**
- 08:00 → 执行"早上开机播放"操作链
- 19:00 → 执行"晚上关机"操作链

## 故障排除

### 问题1：上传失败，显示 "error: espcomm_open failed"

**解决方案：**
1. 检查 USB 线是否支持数据传输（有些线只支持充电）
2. 安装 CH340/CP2102 驱动程序
3. 按住 FLASH 按钮，然后按一下 RESET 按钮进入烧录模式
4. 重新选择正确的 COM 端口

### 问题2：红外学习失败，无法接收信号

**解决方案：**
1. 检查红外接收头的引脚连接是否正确
2. 确保遥控器发射端对准接收头，距离在 5-30 厘米
3. 检查遥控器电池是否有电
4. 在串口监视器中查看是否有原始信号输出

### 问题3：WiFi 连接失败

**解决方案：**
1. 检查 WiFi 名称和密码是否正确（区分大小写）
2. 确保 WiFi 是 2.4GHz 频段（ESP8266 不支持 5GHz）
3. 检查 WiFi 信号强度
4. 尝试重启路由器

### 问题4：Web 界面无法访问

**解决方案：**
1. 确保手机和 ESP8266 连接到同一个 WiFi 网络
2. 检查串口监视器中的 IP 地址是否正确
3. 尝试刷新页面或清除浏览器缓存
4. 检查是否有防火墙阻止连接

### 问题5：操作链或定时任务丢失

**解决方案：**
1. 确保 Flash Size 设置为 4MB (FS:2MB OTA:~1019KB)
2. 检查 LittleFS 是否正常挂载（串口监视器会有提示）
3. 可以重新上传 LittleFS 文件系统

### 问题6：定时任务不准确

**解决方案：**
1. 确保 ESP8266 能正常连接到 NTP 服务器
2. 检查时区设置是否正确（`gmtOffsetSec`）
3. 可以在代码中增加调试输出查看当前时间
4. 考虑使用外部 RTC 模块作为备用时钟源

## API 接口（v3）

| 路径 | 方法 | 功能 |
|------|------|------|
| `/` | GET | 主页（Web 界面） |
| `/status` | GET | 获取系统状态 |
| `/learn/start` | GET | 开始学习红外编码（?name=xxx） |
| `/learn/status` | GET | 查询学习状态 |
| `/codes` | GET | 获取已学习编码列表 |
| `/codes/delete` | GET | 删除编码（?name=xxx） |
| `/send` | GET | 发送红外命令（?name=xxx） |
| `/chains` | GET | 获取操作链列表 |
| `/chains/create` | POST | 创建操作链 |
| `/chains/update` | POST | 更新操作链 |
| `/chains/delete` | GET | 删除操作链（?id=xxx） |
| `/chains/run` | GET | 立即执行操作链（?id=xxx） |
| `/schedules` | GET | 获取定时任务列表 |
| `/schedules/create` | POST | 创建定时任务 |
| `/schedules/update` | POST | 更新定时任务 |
| `/schedules/delete` | GET | 删除定时任务（?id=xxx） |

## 进阶功能

### 接入 Home Assistant

如果你想把这个遥控器接入 Home Assistant，可以：

1. 添加 RESTful Switch 配置：
```yaml
switch:
  - platform: rest
    name: "TV Power"
    resource: http://192.168.1.100/send?name=power
    method: get
```

2. 使用 MQTT 协议（需要额外代码支持）

### OTA 无线固件升级（v3 已内置）

v3 版本已内置 OTA 在线更新功能，无需重新接线即可更新固件：

**使用方法：**
1. 在 Web 界面的状态栏中点击"点击进入 OTA 更新页面"，或直接访问 `http://<设备IP>/update`
2. 选择编译好的 `.bin` 固件文件
3. 点击"Update Firmware"开始升级
4. 等待升级完成，设备会自动重启

**也可以通过 mDNS 访问：**
- 访问 `http://esp8266-ir.local` 进入主界面
- 访问 `http://esp8266-ir.local/update` 进入 OTA 更新页面

**在 Arduino IDE 中导出固件：**
1. 选择 **草图 → 导出已编译的二进制文件**
2. 在项目文件夹中找到 `.bin` 文件
